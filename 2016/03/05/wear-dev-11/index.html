Android Wear 开发 （十一） 同步数据项

Posted on 2016-03-05

DataItem定义了系统用来在手机和手表间同步数据的数据接口。它通常包含了：

  • Payload - 一个字节数组，可以设置任何数据，可以自己进行对象序列化和反序列化。其大小限制在100KB。
  • Path - 一个独特的字符串，必须以一个前斜杠开始，如(/path/to/data)。

一般不需要直接实现DataItem。只需：

  1. 创建一个PutDataRequest对象，指定一个字符串路径来标识该项目。
  2. 调用setData()来设置payload。
  3. 如果同步的延迟会影响用户体验，调用setUrgent()。
  4. 调用DataApi.putDataItem()。

实测表明，不调用setUrgent()的话，手机向手表发送请求时，双方响应非常快。而手表向手机发送时，只有手表处马上响应，手机处则非常慢。

在请求数据项时，系统返回正确实现了DataItem接口的对象。然而，并不推荐使用setData()直接设置原生的字节，建议使用data map，它用一个容易使用的类似Bundle的接口来暴露数据。

用data map同步数据

尽可能使用DataMap类。系统会自动进行对象的序列化和反序列化，像Bundle一样。

  1. 创建PutDataMapRequest对象，为数据项设定路径。

    注意：路径字符串是连接的两侧用来访问数据项的独特的标识符。如果在应用中使用了阶层式的数据，应该创建一个path scheme来匹配数据的阶层。

  2. 调用PutDataMapRequest.getDataMap()来获取一个可以设置值的data map。

  3. 调用put...()方法设置值。
  4. 如果延迟会影响用户体验，调用setUrgent()。
  5. 调用PutDataMapRequest.asPutDataRequest()来获取一个PutDataRequest对象。
  6. 调用DataApi.putDataItem来请求系统创建数据项。

    注意：如果手机和手表断开连接，数据将被缓存，并在连接重新建立后进行同步。
