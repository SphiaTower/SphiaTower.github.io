Recording with AudioRecord APIs in Android

Posted on 2016-03-06

I want to make an audio recorder app for my Android Wear device. In order to get familiar with the recording APIs of the Android platform, I will start with a simple mobile app, and then move it into my moto 360.

APIs

Two sets of audio recording APIs, MediaRecorder and AudioRecord, are available for developing handheld apps. MediaRecorder is much easy to use, however it’s unavailable for wearable devices. So you must process the raw data from AudioRecord by yourself.

Find a Proper Audio Configuration

You must get a proper set of arguments to initialize an AudioRecord object. Different devices support different configurations, however, there’s not any method to get the supported configuration of your device directly. You must query all possible combinations to find an valid one.

Declare Permissions

Don’t forget to declare the permissions. When developing for Marshmellow, remember to require runtime permission, or you can’t create a valid AudioRecord.

<uses-permission android:name="android.permission.RECORD_AUDIO" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

Start Recording

After creating a valid AudioRecord, you can call AudioRecord.startRecording() to start recording. During recording, you should:

  1. Use a flag to indicate that recording has been started.
  2. Write recorded data to file immediately and continuously in background.

Stop Recording

When stopping, don’t forget to mark the flag and call release().

Play Recorded Clips

Most music players on Android could not play a .pcm file. Instead, you should write a simple PCM player using AudioTrack class.
