安卓保持设备唤醒

Posted on 2015-10-01

为了避免耗尽电池，闲置状态的安卓设备，很快会进入睡眠状态。然而，有时应用需要唤醒屏幕或CPU并保持唤醒状态来完成一些工作。

使用的方法取决于应用本身的需求。然而，通用的原则是，应该尽量使用轻量级的方法，来减少应用对系统资源的冲击。下面将探讨当设备的默认睡眠行为无法满足应用需求时的处理方法。

保持屏幕唤醒

某些应用可能需要保持屏幕开启，比如游戏或者电影应用。最好的方法是使用在activity中使用FLAG_KEEP_SCREEN_ON（而且只在activity中使用，永远不要在服务或者其他组件中使用），比如：

public class MainActivity extends Activity {
  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    getWindow().addFlags(WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON);
  }

这个方法的优势是，不像其他唤醒锁一样，不需要特别的权限，而且平台在用户切换应用的过程中正确进行管理，不需要自己来释放资源。

另一个方法是在应用的布局XML中实现，通过使用android:keepScreenOn特性：

<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:keepScreenOn="true">
    ...
</RelativeLayout>

使用android:keepScreenOn="true"等价于使用FLAG_KEEP_SCREEN_ON。在程序中设定的好处是可以在之后清理这个flag，允许屏幕关闭。

除非不需要屏幕继续保持开启，比如希望程序关闭一段时间，不需要去清理这个flag。Window manager会确保应用进入后台或返回前台时正确处理。但如果想要显式地清理flag，并允许屏幕关闭，使用clearFlags()：

getWindow().clearFlags(WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON).

保持CPU唤醒

如果为了在设备睡眠之前完成一些工作，需要保持CPU运转，可以使用PowerManager的一个系统服务，唤醒锁。唤醒锁允许应用控制设备的电源状态。

唤醒和保持唤醒锁可能对电池的生命有严重的影响。所以只有严格需要的使用才可以使用唤醒锁，并保持尽可能短的时间。比如，永远不应该在一个activity中使用唤醒锁。如果想在activity中保持屏幕开启，使用FLAG_KEEP_SCREEN_ON。

一个合理的使用案例是，一个后台服务，需要使用唤醒锁来保持CPU在屏幕关闭后继续工作。尽管如此，这种做法也应该被保持到最小来减少对电池寿命的影响。

要使用唤醒锁，第一步是添加WAKE_LOCK权限：

<uses-permission android:name="android.permission.WAKE_LOCK" />

如果应用包含一个使用服务来工作的broadcast receiver，可以使用WakefulBroadcastReceiver来管理唤醒锁。这是推荐的做法，如果不这样做，可以如下直接设置唤醒锁：

PowerManager powerManager = (PowerManager) getSystemService(POWER_SERVICE);
WakeLock wakeLock = powerManager.newWakeLock(PowerManager.PARTIAL_WAKE_LOCK,
        "MyWakelockTag");
wakeLock.acquire();

要释放唤醒锁，调用wakelock.release()。可以释放对CPU的需求。尽可能快地释放唤醒锁非常重要。

使用WakefulBroadcastReceiver

使用boradcast receiver与service来管理后台任务的生命周期。

WakefulBroadcastReceiver是一种特殊类型的broadcast receiver，负责创建和管理一个PARTIAL_WAKE_LOCK。一个WakefulBroadcastReceiver传递工作到Service（一般是IntentService），同时确保设备在转换过程中不会睡眠。如果不在转换过程中持有唤醒锁，将会允许设备在任务完成前进入睡眠。结果可能导致应用没能及时完成工作。

使用WakefulBroadcastReceiver的第一步是添加至manifest中，和其他broadcast receiver一样：

<receiver android:name=".MyWakefulReceiver"></receiver>

下面的代码用startWakefulService()启动MyIntentService。这个方法类似startService()，除了WakefulBroadcastReceiver在服务启动时保有唤醒锁。传给startWakefulService()的intent持有一个extra来identify唤醒锁：

public class MyWakefulReceiver extends WakefulBroadcastReceiver {

    @Override
    public void onReceive(Context context, Intent intent) {

        // Start the service, keeping the device awake while the service is
        // launching. This is the Intent to deliver to the service.
        Intent service = new Intent(context, MyIntentService.class);
        startWakefulService(context, service);
    }
}

服务结束之后，调用MyWakefulReceiver.completeWakefulIntent()来释放唤醒锁。completeWakefulIntent()方法拥有一个与WavefulBroadcastReceiver相同的intent参数。
