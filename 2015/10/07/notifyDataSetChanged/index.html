A Problem about notifyDataSetChanged()

Posted on 2015-10-07

Problem

I found a problem when using adapter of a RecyclerView. The notifyDataSetChanged() method did not work because I wrapped it in a recursive method which never returns.

And I found that the notifyDataSetChanged() method could only work after the wrapped method returned. For example, if a sleep is added after notifyDataSetChanged(), it won’t take effect before the sleep is finished.

void wrapper(){
    mAdapter.notifyDataSetChanged();
    try{
        TimeUnit.Seconds.sleep(5);
    }catch(Exception ignored){
    }
}

The notifyDataSetChanged() will work 5 seconds later after the wrapper() is called.

Surmise ‘N Solution

The reason why it doesn’t take effect until the method returns may be related with the MessageQueue of UI thread. I guess the notifyDataSetChanged() posts a message into the MessageQueue of UI thread. But if the current method, which running in the UI thread, does not return, the MessageQueue won’t start to handle the message of notifyDataSetChanged(). So maybe this problem could be fixed to use a Handler:

void wrapper(){
    mAdapter.notifyDataSetChanged();
    mHandler.post(new Runnable(){
            try{
                TimeUnit.Seconds.sleep(5);
            }catch(Exception ignored){
            }
        }
    );
}

Then the sleep code won’t block the notifyDataSetChanged(), it will take effect once be called.
