Volley解析UTF-8产生乱码的解决方法

Posted on 2015-10-03

在使用Volley的StringRequest获取字符串的时候遇到了一个问题，返回的中文部分全部都是乱码，查找了一下原因，在这篇文章中找到了答案：

如果在服务器的返回数据的header中没有指定字符集那么就会默认使用 ISO-8859-1 字符集。
ISO-8859-1的别名叫做Latin1。这个字符集支持部分是用于欧洲的语言，不支持中文。

而出问题的代码在StringRequest的这个方法中：

@Override
protected Response<String> parseNetworkResponse(NetworkResponse response) {
    String parsed;
    try {
        parsed = new String(response.data, HttpHeaderParser.parseCharset(response.headers));
    } catch (UnsupportedEncodingException e) {
        parsed = new String(response.data);
    }
    return Response.success(parsed, HttpHeaderParser.parseCacheHeaders(response));
}

所以只需要把HttpHeaderParser.parseCharset(response.headers)的参数换为"UTF-8"就可以了。如果不想修改源码的话，可以创建一个子类来覆盖这个方法：

public class Utf8StringRequest extends StringRequest {
    public Utf8StringRequest(int method, String url, Response.Listener<String> listener, Response.ErrorListener errorListener) {
        super(method, url, listener, errorListener);
    }

    public Utf8StringRequest(String url, Response.Listener<String> listener, Response.ErrorListener errorListener) {
        super(url, listener, errorListener);
    }

    @Override
    protected Response<String> parseNetworkResponse(NetworkResponse response) {
        String parsed;
        try {
            parsed = new String(response.data, HttpHeaderParser.parseCharset(response.headers, "UTF-8"));
        } catch (UnsupportedEncodingException e) {
            parsed = new String(response.data);
        }
        return Response.success(parsed, HttpHeaderParser.parseCacheHeaders(response));
    }
}
