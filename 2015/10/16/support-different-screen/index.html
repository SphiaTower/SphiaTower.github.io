适配不同屏幕（三） 使用标识符

Posted on 2015-10-16

支持不同屏幕尺寸有以下几种方法：

  • 确保足够地缩放布局来适应屏幕；
  • 根据屏幕配置提供合适的UI布局；
  • 确保正确的布局应用在正确的屏幕；
  • 提供正确尺度的位图。

使用尺寸标识符

尺寸标识符可以为不同的屏幕配置提供合适的布局。比如，一些应用在大屏幕上使用双面板，在小屏幕上使用单面板，可以通过分别在：

  • res/layout-large/main.xml
  • res/lauout/main.xml

中提供两个不同的布局来实现。

使用最小宽度标识符

即使很多设备尺寸被归类为大屏幕，有时也需要进一步区分他们的大小。所以安卓在3.2版本之后引入了”Smallest-width”标识符。

最小宽度标识符可以指定拥有某个最小dp宽度的屏幕。例如，典型的七寸平板有600dp的最小宽度。如果希望在这些屏幕中使用双面板，可以使用sw600dp来替代large尺寸标识符。

res/layout-sw600dp/main/xml

也就是说最小宽度>=600dp的设备将会选择上面的xml文件作为布局。

然而这个方法在3.2版本之前不起作用，所以需要在layout-large中创建一个同样的xml文件。

使用布局别名

因为最小宽度仅仅在3.2版本之后才能使用。所以使用的同时也应该使用原来的抽象尺寸标识符（small, normal, large and xlarge）以兼容早期版本。

为了避免同样xml文件的重复，可以使用别名文件。比如在res/layout/main_twopanes.xml中定义双面板的布局，然后创建下面两个布局：

  • res/values-large/layout.xml:


    @layout/main_twopanes

  • res/values-sw600dp/layout.xml:


    @layout/main_twopanes

这两个文件拥有同样的内容，但并不定义布局。只是将main设置为main_twopanes的别名。因为这些文件有large和sw600dp的选项，各个版本的安卓设备都可以应用这些布局。

使用方向标识符

一些布局无论横向还是纵向都可以良好地显示，然而更多的布局需要进行一些调整。比如在横屏时可以显示双面板，或者某些布局可以相对更宽。

每个布局都定义在res/layout/目录下的XML文件中。要将每个布局分配给不同的屏幕配置，可以使用布局别名进行匹配。定义完可能的布局之后，只需要使用配置标识符来映射这些布局。

using the layout alias technique:

res/values/layouts.xml:

@layout/onepane_with_bar
false

res/values-sw600dp-land/layouts.xml:

@layout/twopanes
true

res/values-sw600dp-port/layouts.xml:

@layout/onepane
false

res/values-large-land/layouts.xml:

@layout/twopanes
true

res/values-large-port/layouts.xml:

@layout/twopanes_narrow
true

使用Nine-patch位图

为了支持不同的屏幕尺寸，图片资源同样应该适配不同的尺寸。比如按钮的背景必须适应不同的按钮形状。

如果在尺寸可变的空间上使用简单的图片，图片可能会被uniformly拉伸或缩小。通过使用一种特殊格式的PNG文件，nine-patch位图，可以指明那些区域可以或不可以被拉伸。

因此为可变尺寸的空间设计位图时，最好使用nine-patch。使用SDK中tools/目录下的draw9patch工具可以将普通的图片转换成nine-patch。

注意，nine-patch图片必须使用.9.png扩展名，这样系统才能与普通的PNG文件进行区分。
