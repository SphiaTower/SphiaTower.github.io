适配不同屏幕（一） 理解安卓测量单位

Posted on 2015-10-05

术语和概念

屏幕尺寸：实际的物理尺寸。为了方便，所有尺寸被分为四组：small, normal, large和extra-large。

屏幕密度：屏幕的单位物理区域中所包含的像素的量，一般使用dpi（每英尺点数）描述。所有密度被分为六种泛化的密度：low, medium, high, extra-high, extra-extra-high, extra-extra-extra-high。

分辨率：屏幕上物理像素的总数目。支持多屏幕时，应用只需要关心泛化的屏幕尺寸和密度，而不需要关心分辨率。

密度无关像素（dp)：定义UI布局时应该使用的虚拟像素单位，用来以一种密度无关的方式表达布局尺寸或位置。

密度无关像素与160dpi屏幕上的一个物理像素等同，它是medium密度屏幕的基线密度。运行时，系统根据屏幕的实际密度来处理dp单位的变换。从dp至屏幕像素的转换很简单：px = dp * (dpi / 160)。例如，在240dpi屏幕上，1dp=1.5物理像素。

理解各个单位

所以，各个手机，首先有不同的物理尺寸，同时又有不同的屏幕密度dpi（每英尺像素数），他们的组合又带来了不同的像素量。我们平常说的240 x 320分辨率，就是以px为单位的，也就是横向240像素，纵向320像素。像素点越多，可显示的信息量就越大。而dpi就是这些像素在物理空间上的密度，比如宽度1inch，那密度就是240dpi。dpi越高，屏幕的分辨率也就越高。而由dp的定义可以推出dp的单位是px/(px/inch)=inch，所以dp就是一个与inch类似的长度量纲，只是方便在安卓上使用而已。所谓最难理解的dp，其实根本是无需理解的，dp越大，表明物理尺寸越大而已。而dip则是dp的别名。sp则用在字体上。

然而，一些设备的实际的实际dpi与声明dpi并不一致。比如170dpi的屏幕会被归类为160dpi，所以用dp来度量物理尺寸并非完全准确。所以在实际开发时，应该优先使用match_parent和wrap_content。

比如现在有一部分辨率240px x 320px，而实际的物理尺寸为1.5inch x 2inch，那么它横向的dpi就是240px/1.5inch = 160 px/inch。而240px的宽度换算为dp则是240dp。如果我们设置一个长度为120dp的view，这个view的大小将占据屏幕宽度的一半，同样，如果我们设置一个120px的view，其宽度一样是屏幕一半。二者的实际物理尺寸都是0.75inch。

现在我们将同样的应用跑在一部同大小的高分辨率屏幕上，分辨率为480px x 640px，物理尺寸还是1.5inch x 2inch。这时横向的dpi则是320 px/inch.横向宽度换算为dp则是240dp，与上一个屏幕相同。这时，120dp的view仍然占据屏幕大小的一半，而120px的view则只占据了四分之一。物理尺寸上，120dp的view仍占据0.75inch，而后者则是0.375inch。

接下来，我们换一块低分辨率的大屏手机，分辨率240px x 320px，实际物理尺寸3inch x 4inch，横向dpi为80 px/inch。横向宽度换算为dp为480dp。这时120dp的view只占据屏幕的四分之一，因为屏幕扩大了一倍，其物理尺寸实际上与原来相同，都是0.75inch。而120px的view则占据屏幕的一半，宽度为1.5inch。

从这里可以进一步验证刚才的分析，使用dp可以保证控件的物理尺寸在各种设备上一致，而使用px可以保证其使用的像素点数一致。
