LoaderManager Compares References

Posted on 2015-12-20

Today, one of my Loader implementations behaved totally weird after I modified its code. The Loader loaded successfully at the 1st time, but failed on all following attempts. The loadInBackground() method returned, but onLoadFinished() was not called.

The LoaderManager class compares the new loaded data and old retained data by code like if(oldData!=newData) to decide whether to update the data or not. Note that it uses == operator to compare them. In my case, my Loader retains a same TreeMap object and add/update entries into it, resulting that they’re identified as the same reference, so the loader won’t deliver the new result.

To solve this problem, you have to create a new TreeMap and add back all the old data.
