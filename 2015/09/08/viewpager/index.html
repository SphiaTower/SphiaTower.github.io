ViewPager使用总结

Posted on 2015-09-08

ViewPager是非常常用的一个控件，使用起来也还算简单。但是ViewPager使用时会经常出现一些细微的问题，所以在此总结一下ViewPager的相关内容。

实现滑动视图

可以使用ViewPager实现滑动视图，如果每个页面都占满整个布局，可以在xml中如下表示：

<?xml version="1.0" encoding="utf-8"?>
<android.support.v4.view.ViewPager
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/pager"
    android:layout_width="match_parent"
    android:layout_height="match_parent" />

要插入每个页面的子视图，需要把布局与PagerAdapter进行挂钩，有两种PagerAdapter可供使用：

FragmentPagerAdapter

页面数量较少且固定时使用。

FragmentStatePagerAdapter

页面数量不固定时使用。用户切换页面后，会摧毁fragment，最小化内存使用。

FragmentPagerAdapter会在滑动过后detach fragment，同时将之保留在内存中，可以在重新切换回来后迅速加载。如果有大量的fragment，可以考虑使用FragmentStatePagerAdapter，它会移除掉切换掉的fragment，代价是重新切换回来后需要重建。当fragment数量较少，本身较复杂时推荐使用FragmentPagerAdapter，数量较多时推荐后者。

下面是使用后者的例子：

public class CollectionDemoActivity extends FragmentActivity {
    // When requested, this adapter returns a DemoObjectFragment,
    // representing an object in the collection.
    DemoCollectionPagerAdapter mDemoCollectionPagerAdapter;
    ViewPager mViewPager;

    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_collection_demo);

        // ViewPager and its adapters use support library
        // fragments, so use getSupportFragmentManager.
        mDemoCollectionPagerAdapter =
                new DemoCollectionPagerAdapter(
                        getSupportFragmentManager());
        mViewPager = (ViewPager) findViewById(R.id.pager);
        mViewPager.setAdapter(mDemoCollectionPagerAdapter);
    }
}

// Since this is an object collection, use a FragmentStatePagerAdapter,
// and NOT a FragmentPagerAdapter.
public class DemoCollectionPagerAdapter extends FragmentStatePagerAdapter {
    public DemoCollectionPagerAdapter(FragmentManager fm) {
        super(fm);
    }

    @Override
    public Fragment getItem(int i) {
        Fragment fragment = new DemoObjectFragment();
        Bundle args = new Bundle();
        // Our object is just an integer :-P
        args.putInt(DemoObjectFragment.ARG_OBJECT, i + 1);
        fragment.setArguments(args);
        return fragment;
    }

    @Override
    public int getCount() {
        return 100;
    }

    @Override
    public CharSequence getPageTitle(int position) {
        return "OBJECT " + (position + 1);
    }
}

// Instances of this class are fragments representing a single
// object in our collection.
public static class DemoObjectFragment extends Fragment {
    public static final String ARG_OBJECT = "object";

    @Override
    public View onCreateView(LayoutInflater inflater,
            ViewGroup container, Bundle savedInstanceState) {
        // The last two arguments ensure LayoutParams are inflated
        // properly.
        View rootView = inflater.inflate(
                R.layout.fragment_collection_object, container, false);
        Bundle args = getArguments();
        ((TextView) rootView.findViewById(android.R.id.text1)).setText(
                Integer.toString(args.getInt(ARG_OBJECT)));
        return rootView;
    }
}

在ActionBar中添加tab

使用ActionBar创造tab，需要开启NAVIGATION_MODE_TABS，然后创建几个ActionBar.Tab的实例，并为每个tab实现ActionBar.TabListener。例如下面的代码：

@Override
public void onCreate(Bundle savedInstanceState) {
    final ActionBar actionBar = getActionBar();
    ...

    // Specify that tabs should be displayed in the action bar.
    actionBar.setNavigationMode(ActionBar.NAVIGATION_MODE_TABS);

    // Create a tab listener that is called when the user changes tabs.
    ActionBar.TabListener tabListener = new ActionBar.TabListener() {
        public void onTabSelected(ActionBar.Tab tab, FragmentTransaction ft) {
            // show the given tab
        }

        public void onTabUnselected(ActionBar.Tab tab, FragmentTransaction ft) {
            // hide the given tab
        }

        public void onTabReselected(ActionBar.Tab tab, FragmentTransaction ft) {
            // probably ignore this event
        }
    };

    // Add 3 tabs, specifying the tab's text and TabListener
    for (int i = 0; i < 3; i++) {
        actionBar.addTab(
                actionBar.newTab()
                        .setText("Tab " + (i + 1))
                        .setTabListener(tabListener));
    }
}

如何用ActionBar.TabListener切换tab，取决于如果构建其内容。下面介绍使用ViewPager和fragment组合。

切换tab

要实现用户选择tab时切换ViewPager的页面，可以在ActionBar.TabListener的实现中调用ViewPager的setCurrentItem()方法来选择合适的页面：

@Override
public void onCreate(Bundle savedInstanceState) {
    ...

    // Create a tab listener that is called when the user changes tabs.
    ActionBar.TabListener tabListener = new ActionBar.TabListener() {
        public void onTabSelected(ActionBar.Tab tab, FragmentTransaction ft) {
            // When the tab is selected, switch to the
            // corresponding page in the ViewPager.
            mViewPager.setCurrentItem(tab.getPosition());
        }
        ...
    };
}

同样，通过手势切换后也应该选中响应的tab。通过实现ViewPager.OnPagerChangeListener可以做到这点。比如：

@Override
public void onCreate(Bundle savedInstanceState) {
    ...

    mViewPager = (ViewPager) findViewById(R.id.pager);
    mViewPager.setOnPageChangeListener(
            new ViewPager.SimpleOnPageChangeListener() {
                @Override
                public void onPageSelected(int position) {
                    // When swiping between pages, select the
                    // corresponding tab.
                    getActionBar().setSelectedNavigationItem(position);
                }
            });
    ...
}

使用Title Strip实现滚动标签

如果想显示一部分的标签，并允许滚动，可以使用PageTitleStrip来实现。
下面的例子中，Activity的整个页面被ViewPager占满，而PagerTitleStrip在里面的最上方，ViewPager里剩下的空间被adapter提供的页面占据。

<android.support.v4.view.ViewPager
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/pager"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <android.support.v4.view.PagerTitleStrip
        android:id="@+id/pager_title_strip"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:layout_gravity="top"
        android:background="#33b5e5"
        android:textColor="#fff"
        android:paddingTop="4dp"
        android:paddingBottom="4dp" />

</android.support.v4.view.ViewPager>

PagerTitleStrip

PagerTitleStrip非交互式地指示ViewPager的当前，前面和后面的页面。在xml中作为ViewPager的子view使用。设置android:layout_gravity为TOP或BOTTOM可以把它固定在ViewPager的顶端或底端。每一页的标题通过adapter的getPageTitle(int)方法获得。

PagerTabStrip

PagerTabStrip是PagerTitleStrip的子类，与之不同的是，增加一些交互效果。

经常发生的一些问题

  • 修改缓存页面的数量。只有三四个页面时这点尤为重要。默认设置存储当前页面两边的两个页面。总共有三个页面时，滑动到中间的页面，所有页面都会被缓存。而滑动到第一个或者最后一个页面时，其中一个页面会掉出内存，再次切换到的时候需要被重建，然后重新添加。通过设置setOffscreenPageLimit(2)可以让所有页面都保留在内存中。这样就可以牺牲内存占用来换取表现效果。要注意内存不足时产生的警告，必要时移除边缘的页面。
  • 替换ViewPager中的某个页面，仅仅改变adapter的数据然后调用notifyDataSetChanged()还不够，还需要确保getItemPosition(Object object)正确实现，替换掉的子项返回POSITION_NONE，没有则返回POSITION_UNCHANGED或者实际位置。
  • 用setPageMargin()和setPageMarginDrawable()可以轻松地分离页面。
