安卓内存泄露问题（一） Handler

Posted on 2015-09-06

本篇文章探讨安卓中内部类和Handler引发的Context泄露问题，文章主要翻译和总结自
http://www.androiddesignpatterns.com/2013/01/inner-class-handler-memory-leak.html

首先来看以下的代码：

public class SampleActivity extends Activity {

  private final Handler mLeakyHandler = new Handler() {
    @Override
    public void handleMessage(Message msg) {
      // ... 
    }
  }
}

上面的代码会隐蔽地造成严重的内存泄露。安卓lint会给出警告：

安卓中，Handler类必须为静态类，否则可能发生泄露
泄露发生在哪里？为了解决这个问题，首先记录下目前已知的条件：

  1. 应用启动时，framework首先在主线程创建一个Looper对象。Looper实现了一个简单的消息队列，在循环中一个接一个地处理Message对象。所有主要的应用框架事件（如Activity的生命周期方法调用，按钮点击等）都包含在Message对象中，它们加入到Looper的消息队列中，依次处理。主线程的Looper存活在整个应用的生命周期中。
  2. 在主线程中初始化的Handler，与Looper的消息队列相关联。添加到队列中的消息持有Handler的引用，以便framework可以在Looper处理完消息后调用Handler#handleMessage(Message)方法。
  3. Java中，非静态的内部类和匿名内部类隐式地持有外部类的引用。静态内部类则不持有。

所以很难找到内存泄露发生的地方。接下来，参考下面的代码： 

public class SampleActivity extends Activity {

  private final Handler mLeakyHandler = new Handler() {
    @Override
    public void handleMessage(Message msg) {
      // ...
    }
  }

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);

    // Post a message and delay its execution for 10 minutes.
    mLeakyHandler.postDelayed(new Runnable() {
      @Override
      public void run() { /* ... */ }
    }, 1000 * 60 * 10);

    // Go back to the previous Activity.
    finish();
  }
}

在Acitivity终止后，延迟的消息在被处理之前，会继续在主线程的消息队列中存活10分钟。而这个消息持有Handler的引用，Handler则隐式持有其外部类（SampleActivity）的引用。这个引用会在消息被处理之前一直维持着，阻止activity的context被gc回收，导致了应用资源的泄露。
上面代码中的匿名Runnable类也是如此。匿名内部类作为非静态类也持有外部类的隐式引用，所以会产生context的泄露。
为了解决这个问题，在继承Handler类时，应选择在新的文件中创建外部类或者使用静态内部类。静态内部类不持有外部类的引用，不会泄露activity。那么如何在不泄露context的情况下调用外部activity的方法呢？答案是让handler持有一个activity的弱引用WeakReference。为了解决创建匿名Runnable时产生的内存泄露，我们同样也让Runnable变量作为类的一个静态域存在（因为匿名内部类的静态实例不持有外部类的隐式引用。

public class SampleActivity extends Activity {

  /**
   * Instances of static inner classes do not hold an implicit
   * reference to their outer class.
   */
  private static class MyHandler extends Handler {
    private final WeakReference<SampleActivity> mActivity;

    public MyHandler(SampleActivity activity) {
      mActivity = new WeakReference<SampleActivity>(activity);
    }

    @Override
    public void handleMessage(Message msg) {
      SampleActivity activity = mActivity.get();
      if (activity != null) {
        // ...
      }
    }
  }

  private final MyHandler mHandler = new MyHandler(this);

  /**
   * Instances of anonymous classes do not hold an implicit
   * reference to their outer class when they are "static".
   */
  private static final Runnable sRunnable = new Runnable() {
      @Override
      public void run() { /* ... */ }
  };

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);

    // Post a message and delay its execution for 10 minutes.
    mHandler.postDelayed(sRunnable, 1000 * 60 * 10);

    // Go back to the previous Activity.
    finish();
  }
}

静态和非静态内部类的区别并不明显，但却是安卓开发者必须掌握的。关键是：在activity中，如果内部类实例的存活时间可能超过activity的生命周期，那么避免使用非静态的内部类。优先选择静态内部类，并在其中持有activity的弱引用。
