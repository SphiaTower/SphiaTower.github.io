RecyclerView添加Divider

Posted on 2015-09-10

与ListView不同，RecyclerView的默认实现没有Divider，需要手动添加。自己以前的做法，是把item的最外层View重写，用canvas画图来实现divider。这种做法从解耦合的角度上非常差，而且经常会出现一些莫名其妙画不出来的问题。所以在这里好好研究一下为RecyclerView添加分割线的方法。

ItemDecoration

首先我们注意到RecyclerView有一个addItemDecoration(ItemDecoration)的方法，方法的描述是ItemDecoration可以影响item view的测量和绘制。接下来研究一下这个类。

Class Overview

一个ItemDecoration可以为特定的item view添加特殊的绘制和布局偏置。可以用来绘制分隔线，高亮，分组边界等等。
所有的ItemDecoration按照添加的顺序来绘制，在item view之前（onDraw()），在item view之后（onDrawOver(Canvas, RecyclerView, RecyclerView.State)）。

Sample

所以，可以看出这个类的使用还是比较简单的。下面是网络上找到的一个实现的例子：

public class DividerItemDecoration extends RecyclerView.ItemDecoration {
    private static final int[] ATTRS = new int[]{android.R.attr.listDivider};
    private Drawable mDivider;

    public DividerItemDecoration(Context context) {
        final TypedArray styledAttributes = context.obtainStyledAttributes(ATTRS);
        mDivider = styledAttributes.getDrawable(0);
        styledAttributes.recycle();
    }

    @Override
    public void onDraw(Canvas c, RecyclerView parent, RecyclerView.State state) {
        int left = parent.getPaddingLeft();
        int right = parent.getWidth() - parent.getPaddingRight();
        int childCount = parent.getChildCount();
        for (int i = 0; i < childCount; i++) {
            View child = parent.getChildAt(i);
            RecyclerView.LayoutParams layoutParams = (RecyclerView.LayoutParams) child.getLayoutParams();
            int top = child.getBottom() + layoutParams.bottomMargin;
            int bottom = top + mDivider.getIntrinsicHeight();
            mDivider.setBounds(left, top, right, bottom);
            mDivider.draw(c);
        }
    }
}

代码很简单，就是在每个item的底部绘制一个drawable而已。值得注意的是，与item的位置相结合，用这个类可以实现很多复杂的效果，让列表显示的元素有更多的特性。以后用到了会总结出来（比如到达最后一页等等）。

上面的代码有一点问题，就是adapter还没有绑定数据时，会出现一个多余的分隔符（原本是整片的空白）。解决方法就是把循环的结束变为childCount-1即可。
