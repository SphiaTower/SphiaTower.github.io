安卓动画学习笔记（三） 属性动画

Posted on 2015-09-13

学习了上两节发现其中的代码有很多看不懂的地方，所以接下来从动画本身来学起。

属性动画是一个几乎能为任何东西实现动画的健壮框架。可以定义动画来随时间改变对象属性，无论是否绘制在屏幕上。属性动画在一段时间内改变一个属性（对象的域）。必须指定想要进行动画的对象属性，比如其位置，动画的时长，动画起止的属性值。
属性动画系统允许设定以下几个特性：

  • 时长：可以指定动画的时长。默认长度为300ms。
  • 时间插值：指定属性值如何随时间流逝变化。
  • 重复计数和行为：可以指定当动画结束后是否重复，以及重复的次数。也可以指定是否倒放动画。倒放可以使动画正向反向反复播放，直到达到次数。
  • 动画集合：可以将动画分组为逻辑集合，或一同播放，或按次序播放，或在一定延时后播放。
  • 帧刷新延迟：可以指定动画刷新帧的频率。默认设置是每10ms刷新一次，但实际的刷新率是由系统的繁忙程度和响应计时器的速度决定的。

属性动画如何工作

假如一个对象以x属性进行动画，在屏幕中横向移动。动画的时长为40ms，移动的距离为40px。每10ms（默认的刷新率），对象横向移动10px。在40ms的最后，动画停止，对象停留在40px的位置。这个动画使用的是线性插值，也就是说，对象匀速移动。
可以指定动画使用非线性插值。比如在开始时加速，在结束时减速。对象仍然在40ms中移动40px，但是非线性的。开始，动画加速到一半的距离，然后减速到终点。前一段时间和最后一段时间移动的距离比中间部分要短一些。

ValueAnimator封装了定义动画插值的TimeInterpolator和定义如何计算属性值的TypeEvaluator。例如上面的第二个例子中，使用AccelerateDecelerateInterpolator来插值，使用IntEvaluator来计算属性值。
要启动一段动画，首先创建一个ValueAnimator，给它动画起止的属性值，与动画的时长。调用start()后动画开始。整个过程中，ValueAnimator根据整个时长和已过去的时间，计算流逝比例（0和1之间）。流逝比例代表动画完成的百分比。比如上面的例子中，10ms时的流逝比例为0.25。
ValueAnimator计算完流逝比例后，调用当前设置 的TimeInterpolator计算插值比例。插值比例将流逝比例映射到一个考虑了时间插值的新的比例。例如，在变速运动的的例子中，因为动画开始缓慢加速，在10ms处，插值比例大概为0.15，小于流逝比例0.25.而匀速运动时，二者一直相等。
插值比例计算过后，ValueAnimator调用合适的TypeEvaluator，根据插值比例，起始值，终止值来计算属性值。在变速运动的例子中，10ms处的插值比例为0.15，所以属性值为0.15*(40-0)=6.

属性动画与视图动画的不同

视图动画系统仅仅支持View对象，如果要动画其他对象，必须用自己的代码实现。视图动画系统另外的局限是，它仅仅暴露了View对象的几个方面，比如尺寸和旋转，但不支持背景颜色。
视图动画系统的另一个缺点是它仅仅改变了View绘制的位置，而非其本身。例如，移动一个按钮时，绘制的位置改变了，但实际点击时的位置却没有变，必须自己实现逻辑来处理这个问题。
属性动画系统不被这些限制束缚。动画可以应用到任何对象，而且对象本身被真正地修改。属性动画执行时也更加健壮。在更高的层次上，给想要动画的属性分配animator，比如颜色，位置，尺寸，并且可以定义插值方式和多个animator的同步。
然而，视图动画系统配置更简单，需要更少的代码。如果视图动画足够完成工作，或者现存的代码已经够用，将不需要用属性动画系统。

API Overview

在android.animation中可以找到大多数的属性动画。而视图动画系统在android.view.animaion中已经定义了许多插值器，在属性动画系统中也可以使用。
Animator类提供了创建动画的基本结构。一般不会直接使用这个类，因为它仅仅提供了支持属性动画的最小限度的功能。下面是它的一些子类：

  • ValueAnimator：属性动画的主要计时引擎，也计算了动画的属性值。它也含有计算动画值的所有核心功能，以及动画计时的细节，是否重复，接收更新事件的侦听器，设置自定义类型来求值的能力。属性动画有两个部分：一是计算属性值，二是将属性值设置到对象上。ValueAnimator不包含第二个部分，所以必须侦听ValueAnimator计算得到属性值的更新，并且实现自己的逻辑来修改对象。
  • ObjectAnimator：ValueAnimator的子类，允许设置一个目标对象和目标属性来进行动画。在动画中计算出新的值后会更新属性值。这个类更加常用一些，因为改变目标对象的属性值更加容易一些。但它也有一些限制，比如要求目标对象必须存在指定的访问方法。
  • AnimatorSet：提供了将一些动画组合使用的机理，使动画可以根据其他动画相对运行。

Evaluator告诉属性动画系统如何计算属性值。它从Animator类中得到计时数据，动画的起始值和终止值，计算属性值。

  • IntEvaluator：计算int属性的默认求值器。
  • FloatEvaluator：计算float属性的默认求值器。
  • ArgbEvaluator：计算十六进制颜色属性的默认求值器。
  • TypeEvaluator：可以创建自定义求值器的接口。如果动画的属性不是int, float或者颜色，就必须实现这个接口来指定如何计算属性值。也可以为上面提到的类型实现自定义的TypeEvaluator接口。

时间插值器定义动画中指定的值如何随时间变化。

  • AccelerateDecelerateInterpolator：开始和结束时速率较慢，在中间加速。
  • AccelerateInterpolator：开始速度较慢，然后加速。
  • AnticipateInterpolator：开始反向变化，然后迅速回到正向。
  • AnticipateOvershootInterpolator：开始反向变化，然后迅速回到正向，接着超过目标值，在返回到目标值。
  • BounceInterpolator：变化在最后反弹。
  • CycleInterpolator：动画重复指定次数。
  • DecelerateInterpolator：开始快速变化，然后减速。
  • LinearInterpolator：变化率恒定。
  • OvershootInterpolator：迅速正向变化，超越最终值，然后返回。
  • TimeInterpolator：实现自定义插值的接口。

使用ValueAnimator

使用工厂方法ofInt(), ofFloat(), ofObject()可以获得valueAnimator实例。比如：

ValueAnimator animation = ValueAnimator.ofFloat(0f, 1f);
animation.setDuration(1000);
animation.start();

上面的代码运行后，ValueAnimator开始计算动画的值，在0与1之间，长度为1000ms。
也可以提供自定义类型：

ValueAnimator animation = ValueAnimator.ofObject(new MyTypeEvaluator(), startPropertyValue, endPropertyValue);
animation.setDuration(1000);
animation.start();

上面的两段代码没有实际的作用，因为ValueAnimator并不直接操作对象或属性。一般需要根据得到的值来修改想要动画的对象。在ValueAnimator中定义侦听器可以在动画期间合适地处理重要事件，比如帧的更新。实现侦听器时，可以调用getAnimatedValue()来获取计算结果。

使用ObjectAnimator

ObjectAnimator是ValueAnimator的子类，把父类的计时引擎，求值计算与改变目标对象的具体属性的能力相结合。使得具体对象的动画更加容易，无需实现ValueAnimator.AnimatorUpdateListener接口，变动的属性会自动更新。
创建其实例与父类相似，但也可以指定目标对象及其属性的名字（作为字符串）：

ObjectAnimator anim = ObjectAnimator.ofFloat(foo, "alpha", 0f, 1f);
anim.setDuration(1000);
anim.start();

为了让ObjectAnimator能够正确更新属性，必须：

  • 属性必须有一个set<propertyName>形式的setter函数（驼峰命名）。因为ObjectAnimator在动画过程中会自动更新属性，所以必须能够通过setter方法访问到属性。如果不存在的话：
    • 如果拥有权限的话，可以添加一个setter方法。
    • 创建一个wrapper类，用它的setter方法接收值，并转发给原来的对象。
    • 使用ValueAnimator代替。
  • 如果只为工厂方法指定values...中的一个值，那个值会被认为是最终值。因此，对象属性必须提供一个getter函数来获取其起始值。
  • getter（如果需要）和setter方法必须操作同样的类型来作为起始值和终止值。
  • 根据运行动画的属性或者对象，有时可能需要调用View的invalidate()方法来使屏幕强制重绘，更新动画。这些操作在onAnimationUpdate()回调中进行。比如改变一个Drawable对象的颜色属性只在对象重绘时更新在屏幕上。View中所有属性的setter，都会适时地让View无效，所以无需手动更新。
