安卓拍摄和保存照片

Posted on 2015-09-23

在应用中可能会用到拍摄照片的功能，而这部分功能在整个应用中可能只是一小部分，所以开发时没有必要花精力去处理到底如何拍照。幸运的是，可以使用安卓设备中的其他相机应用来帮你处理。

要求相机权限

如果拍照时应用的核心内容，那么需要限制安装应用的手机必须有相机。在manifest中可以如下处理：

<manifest ... >
    <uses-feature android:name="android.hardware.camera"
                  android:required="true" />
    ...
</manifest>

如果应用使用，但是不要求必须拥有相机，可以设置android:required为false。这样即使没有相机也可以使用应用，而在运行时可以通过调用hasSystemFeature(PackageManager.FEATURE_CAMERA)方法来检查是否可以使用相机。

使用相机应用拍摄照片

安卓借助一个描述行为意图的Intent，将行为委托给其他的应用。整个过程中有三个部分：Intent自身，打开外部Activity，回到activity时的处理。

static final int REQUEST_IMAGE_CAPTURE = 1;

private void dispatchTakePictureIntent() {
    Intent takePictureIntent = new Intent(MediaStore.ACTION_IMAGE_CAPTURE);
    if (takePictureIntent.resolveActivity(getPackageManager()) != null) {
        startActivityForResult(takePictureIntent, REQUEST_IMAGE_CAPTURE);
    }
}

注意startActivityForResult()方法由resolveActivity()所进行的条件检查进行保护，后者返回第一个可以处理intent的activity组件。这里的检查非常重要，因为如果没有应用可以处理intent，自己的应用将会崩溃。所以只要返回值非null，就能够安全地使用intent。

获取缩略图

如果应用要做的不仅仅是拍摄照片，可能会需要从相机应用中取回图片，进行处理。
安卓相机应用将照片编码成Bitmap放到Intent中传递给onActivityResult()方法。照片被放在Intent的extras中，其key为"data"。下面的代码取回图片并显示：

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    if (requestCode == REQUEST_IMAGE_CAPTURE && resultCode == RESULT_OK) {
        Bundle extras = data.getExtras();
        Bitmap imageBitmap = (Bitmap) extras.get("data");
        mImageView.setImageBitmap(imageBitmap);
    }
}

注意：从"data"中取回的缩略图，可能仅仅适合作为图标使用。处理全尺寸的图片需要更多的工作。

保存全尺寸照片

如果给安卓相机应用一个文件去存储，那么它将保存完整尺寸的照片。必须提供一个完整的文件名来了告诉相机应用在哪保存照片。
一般，任何用户用相机拍摄的照片，都应该存在设备的公共外部存储中，这样可以使所有应用都可以访问。存储共享的照片的合适路径由getExternalStoragePublicDirectory()和DIRECTORY_PICTURES参数提供。因为这个目录由所有应用共享，读写时分别需要READ_EXTERNAL_STORAGE和WRITE_EXTERNAL_STORAGE权限。写权限隐含了读取权限，所以只需要请求写权限就可以了。

<manifest ...>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
    ...
</manifest>

然而，如果希望想保持照片为应用私有，可以使用getExternalFilesDir()方法提供的目录。安卓4.3及以下版本，写进这个目录需要提供写权限。之后的版本则不需要，因为目录不会被其他应用访问。添加maxSdkVersion，就可以仅仅在低版本中请求写权限。

<manifest ...>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
                     android:maxSdkVersion="18" />
    ...
</manifest>

注意：写进这个目录的文件会在用户卸载时删除。

一旦决定了存储文件的目录，就需要创建一个不冲突的文件名。另外也可能需要就路径放在成员变量中以便以后使用。下面是一个使用日期时间戳为新照片获得一个独一无二的用户名的方法：

String mCurrentPhotoPath;

private File createImageFile() throws IOException {
    // Create an image file name
    String timeStamp = new SimpleDateFormat("yyyyMMdd_HHmmss").format(new Date());
    String imageFileName = "JPEG_" + timeStamp + "_";
    File storageDir = Environment.getExternalStoragePublicDirectory(
            Environment.DIRECTORY_PICTURES);
    File image = File.createTempFile(
        imageFileName,  /* prefix */
        ".jpg",         /* suffix */
        storageDir      /* directory */
    );

    // Save a file: path for use with ACTION_VIEW intents
    mCurrentPhotoPath = "file:" + image.getAbsolutePath();
    return image;
}

使用这个方法就可以为相片创建一个文件，现在可以这样创建和使用Intent：

static final int REQUEST_TAKE_PHOTO = 1;

private void dispatchTakePictureIntent() {
    Intent takePictureIntent = new Intent(MediaStore.ACTION_IMAGE_CAPTURE);
    // Ensure that there's a camera activity to handle the intent
    if (takePictureIntent.resolveActivity(getPackageManager()) != null) {
        // Create the File where the photo should go
        File photoFile = null;
        try {
            photoFile = createImageFile();
        } catch (IOException ex) {
            // Error occurred while creating the File
            ...
        }
        // Continue only if the File was successfully created
        if (photoFile != null) {
            takePictureIntent.putExtra(MediaStore.EXTRA_OUTPUT,
                    Uri.fromFile(photoFile));
            startActivityForResult(takePictureIntent, REQUEST_TAKE_PHOTO);
        }
    }
}

添加照片至相册

通过intent创造的照片，其存储地址是已知的，因为一开始就确定了存储的位置。但对用户来说，访问照片的最简单的方式是让照片可以通过系统的Media Provider访问。

注意：如果使用getExternalFilesDir()目录，媒体扫描器将无法访问文件，因为他们是应用私有的。

下面的例子展示了如何调用系统的媒体扫描器来把照片添加到Media Provider的数据库，并呈现在安卓的媒体库应用和其它应用中。

private void galleryAddPic() {
    Intent mediaScanIntent = new Intent(Intent.ACTION_MEDIA_SCANNER_SCAN_FILE);
    File f = new File(mCurrentPhotoPath);
    Uri contentUri = Uri.fromFile(f);
    mediaScanIntent.setData(contentUri);
    this.sendBroadcast(mediaScanIntent);
}

解码压缩后的图片

在有限的内存中管理全尺寸的图片很可能出现问题。如果在显示了仅仅几张图片之后就发现应用耗尽内存，就应该将JPEG压缩到目标view匹配的大小后再展开至内存中，以大量减少动态堆的使用。

private void setPic() {
    // Get the dimensions of the View
    int targetW = mImageView.getWidth();
    int targetH = mImageView.getHeight();

    // Get the dimensions of the bitmap
    BitmapFactory.Options bmOptions = new BitmapFactory.Options();
    bmOptions.inJustDecodeBounds = true;
    BitmapFactory.decodeFile(mCurrentPhotoPath, bmOptions);
    int photoW = bmOptions.outWidth;
    int photoH = bmOptions.outHeight;

    // Determine how much to scale down the image
    int scaleFactor = Math.min(photoW/targetW, photoH/targetH);

    // Decode the image file into a Bitmap sized to fill the View
    bmOptions.inJustDecodeBounds = false;
    bmOptions.inSampleSize = scaleFactor;
    bmOptions.inPurgeable = true;

    Bitmap bitmap = BitmapFactory.decodeFile(mCurrentPhotoPath, bmOptions);
    mImageView.setImageBitmap(bitmap);
}

手动存储图片

上面存储图片的例子是使用Intent来自动处理的，那么怎么收到存储一个图片呢？
使用MediaStore就可以了：

MediaStore.Images.Media.insertImage(ContentResolver cr, Bitmap source, String title, String description)

这个方法可以将图片存到默认路径下，那么如果想将图片存到其他路径，或者进行压缩处理呢？

try {
    file = createImageFile();
    Bitmap bitmap = getBitmap();
    FileOutputStream os = new FileOutputStream(file);
    bitmap.compress(Bitmap.CompressFormat.JPEG, 100, os);
    os.close();
    MediaStore.Images.Media.insertImage(getContentResolver(),file.getAbsolutePath(),file.getName(),file.getName());
} catch (IOException e) {
    e.printStackTrace();
    Toast.makeText(getActivity(), "failed to save the image", Toast.LENGTH_SHORT).show();
}

使用重载的insertImage()方法就可以将图片存至其他指定路径了。
