Java 8泛型方法的自动推导

Posted on 2015-09-23

今天在一个工程中写下面的语句时产生了疑问：

imageView.setImageBitmap(getIntent().getExtras().getParcelable());

编译器居然没有对这个语句的类型错误进行提醒。setImageBitmap()接收一个Bitmap对象，而getParcelable()方法返回的应该是一个Parcelable对象。其中Bitmap类实现了Parcelable接口，所以这里存在一个向下转型，理应出现类型转换的问题，却没有报错。这是为什么呢？

原因一是工程中用了Java 8的插件，并将项目的语言level提升到了Java 8，二是getParcelable()方法返回的并非是Parcelable对象本身。下面是方法的源码：

public <T extends Parcelable> T getParcelable(String key) {
    unparcel();
    Object o = mMap.get(key);
    if (o == null) {
        return null;
    }
    try {
        return (T) o;
    } catch (ClassCastException e) {
        typeWarning(key, o, "Parcelable", e);
        return null;
    }
}

这个方法，根据泛型将取得的Parcelable对象转换成了其子类。而在上面的代码中，并没有在调用方法时注明泛型子类，所以理应会出现类型错误。但在JEP101中，引入了泛型方法的自动推导。返回得到的对象会被自动转型成为setImageBitmap()方法签名中的Bitmap类。这样就不需要手动注明泛型参数了。同样的代码在java8以前的版本需要这样写：

imageView.setImageBitmap(getIntent().getExtras().<Bitmap>getParcelable());

另外需要注明的一点是，使用Intent来传递Bitmap时，bitmap的大小必须很小才可以，所以一般不能通过这样方式在activity之间传递Bitmap。
