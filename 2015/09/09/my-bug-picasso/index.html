Picasso获取异常信息与正则匹配时的一个低级错误

Posted on 2015-09-09

我的正在做的一个应用里，有一部分需要获取一段字符串，并解析其中的特殊格式标签，以获取其中的文字和图片。一开始使用了正则表达式对标签做了简单的处理，转换成html标签，然后使用Html.fromHtml()获取其中的图片（与文字混排）。今天决定换成使用Picasso实现时，出现了一个错误纠结了很长时间才解决。代码很简单： 

Picasso.with(getContext()).load(url).into(imageView);

其中的url是通过正则匹配获得的。
运行时一直获取不到图片，imageView上显示error状态的占位图。在网络上搜索了一下，找到了获取error信息的方法：

Picasso picasso = new Picasso.Builder(getContext()).listener(new Picasso.Listener() {
    @Override
    public void onImageLoadFailed(Picasso picasso, Uri uri, Exception exception) {
        exception.printStackTrace();
    }
}).build();
picasso.load(url).into(imageView);

这样就可以获取error的异常信息了。获得的信息主要就是：

java.lang.IllegalStateException: Unrecognized type of request: Request{"http://www.abc.com/def.jpg"}

而点击其中的链接，获取图像毫无问题。将上面的url复制粘贴后，直接创建一个String url常量，也毫无问题。最后用Java比较了两个字符串变量才发现问题：原来匹配的时候多带了两边的引号！在原来的实现中，放入html标签没有问题；而换了Picasso之后，当然不能再带上引号了。
这个错误虽然很低级，但的确很难发现，特别是从原来版本的实现换成Picasso时，很容易忽略这样的问题。
