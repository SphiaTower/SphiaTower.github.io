创造自定义View并在XML中使用自定义特性

Posted on 2015-09-20

好的自定义view应该将功能封装为方便的接口，高效使用CPU和内存。另外还要做到：

  • 适应安卓标准
  • 提供自定义的风格属性，兼容安卓的XML布局
  • 发送无障碍事件
  • 兼容多个安卓平台

继承View

安卓框架中，所有view都继承自View类。
为了允许安卓开发者工具与view交互，至少需要提供一个接收Context和AttributeSet对象的构造器。这样布局编辑器就可以创造和编辑这个view了。

class PieChart extends View {
    public PieChart(Context context, AttributeSet attrs) {
        super(context, attrs);
    }
}

定义自定义特性

要想让自定义view能够在XML中添加和编辑，需要：

  • 在<declare-styleable>资源元素中定义自定义特性。
  • 在XML布局中指定这些特性的值
  • 在运行时取回这些特性的值
  • 为自定义view应用这些值

这一小节首先来看如果定义和指定自定义特性及其值，下一小节来看如果在运行时取回和应用这些值。
定义自定义特性，需要在res/values/attrs.xml中添加<declare-styleable>资源。

<resources>
   <declare-styleable name="PieChart">
       <attr name="showText" format="boolean" />
       <attr name="labelPosition" format="enum">
           <enum name="left" value="0"/>
           <enum name="right" value="1"/>
       </attr>
   </declare-styleable>
</resources>

上面的代码声明了Piechart条目中的两个自定义特性showText和labelPosition，风格条目的名字一般与自定义类的名字相同。
定义之后，就可以在XML中像内置特性一样使用了。自定义特性的命名空间与内置的不同，它们属于http://schemas.android.com/apk/res/[your package name]而非http://schemas.android.com/apk/res/android，请看下面的例子：

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
   xmlns:custom="http://schemas.android.com/apk/res/com.example.customviews">
 <com.example.customviews.charting.PieChart
     custom:showText="true"
     custom:labelPosition="left" />
</LinearLayout>

为了避免重复冗长的命名空间URI，这个例子使用xmlns指令。这个指定为命名空间http://schemas.android.com/apk/res-auto分配了custom别名。

注意：如果没有使用gradle，xlmnsURI不能包含res-auto，必须包含项目的全名。如http://schemas.android.com/apk/res/com.example.customviews.

注意layout中自定义view的XML标签的名字是自定义view的全名。如果该类是内部类，必须提供外部类的名字。例如，PieChart类是PieView的内部类的话，就需要使用com.example.customviews.charting.PieChart$PieView作为标签。

应用自定义特性

当一个view通过XML布局创建时，所有XML标签中的特性从资源bundle中读取，并作为AttributeSet参数传递给view的构造器。虽然可以直接从AttributeSet中读取，但这样有一些缺点：

  • 特性值中的资源引用未能解析
  • 风格没能应用

最好传递AttributeSet给obtainStyledAttributes()。这个方法传回一个解除引用和风格化的TypedArray。
安卓资源编译器为了使调用obtainStyledAttributes()更加简单做了很多工作。为每个<declare-styleable>资源，生成的R.java都定义了一列特性id和每个特性的索引的常数集合。可以使用这些预先定义的常数来从TypedArray中读取常数。

public PieChart(Context context, AttributeSet attrs) {
   super(context, attrs);
   TypedArray a = context.getTheme().obtainStyledAttributes(
        attrs,
        R.styleable.PieChart,
        0, 0);

   try {
       mShowText = a.getBoolean(R.styleable.PieChart_showText, false);
       mTextPos = a.getInteger(R.styleable.PieChart_labelPosition, 0);
   } finally {
       a.recycle();
   }
}

注意TypedArray对象是共享资源，必须在使用后回收。

添加属性和事件

特性是一种控制view行为和外观的强大工具，但只能在view创建时读取。为了提供动态的行为，可以为每个自定义特性暴露getter和setter。

public boolean isShowText() {
   return mShowText;
}

public void setShowText(boolean showText) {
   mShowText = showText;
   invalidate();
   requestLayout();
}

注意setShowText调用了invalidate()和requestLayout()。这些调用时view正确行为的关键。在改变了任何可能影响外观的属性后，都必须invalidate这个view，这样系统就知道它需要被重绘。如果一个属性变化可能影响view的大小和形状，则需要请求一个新的布局。
自定义view应该支持事件侦听器来与重要的事件进行通讯。例如，PieChart暴露了一个自定义事件OnCurrentItemChanged来通知侦听器，用户旋转了饼状图，以将焦点转移到新的pie片段。
很容易忘记暴露属性和时间，尤其是这个view只有你自己使用的时候。花一些时间来仔细定义view的接口可以减少未来维护的消耗。一个好的原则是：总是暴露影响行为或外观的属性。

无障碍设计

自定义view应该支持更多的人群，包括残疾人士。要做到这一点应该：

  • 通过android:contentDescription特性为输入的控件注明标签
  • 合适的时候通过sendAccessibilityEvent()发送无障碍事件
  • 支持一些替代的控制器，比如D-pad和trackball.
