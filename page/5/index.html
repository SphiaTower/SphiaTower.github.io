An Encoding Problem in Clion

Posted on 2015-11-11

Clion is a great IDE of C/C++, made by JetBrain. But I found a problem that when the output of console is Chinese, it can’t be displayed correctly.

I tried to use the same method as Intellij IDEA, changing the encoding from GBK to UTF-8 at the right bottom corner. However, it didn’t work like IDEA 14.

After a lot of attempts modifying different settings, I have found the solution: In File > Settings > Editor > File Encoding, modify both IDE Encoding and Project Encoding into UTF-8. After applied, click the console dock and then change the right bottom corner from UTF-8 to GBK. Then the console can output Chinese now.

EmptyView in ViewPager

Posted on 2015-10-15

Today I encountered a very subtle bug that the Fragments restored from a ViewPager show its empty view with their data already loaded.

My CustomFragment shows an empty view when initialized, and switch to the RecyclerView once data is loaded. The process of animation has been summarized in one of my old articles(http://sphiatower.github.io/2015/09/08/viewpager/). In short, it hides the RecyclerView and shows the empty view in the onCreateView callback.

Every time the data of RecyclerView is updated, it will check an empty flag whether it’s the first time of loading data. If true, the cross fade animation is played to show the content and the flag is updated to remember that data has been loaded.

安卓文件存储总结

Posted on 2015-10-14

安卓使用的文件系统与其他平台的磁盘文件系统相似。这里总结如何使用FileAPI读写文件。

File对象适合一次性读写大量数据，如图片文件或其他来自网络的数据。

选择内部或外部存储

所有安卓设备都有两个存储区域：内部存储和外部存储。这些名称来自安卓的早期时候，当时大多数设备即提供了内置的固定存储，也提供了可移动的存储媒介（比如SD卡）。一些设备将永久存储空间分为内部和外部两部分，所以即使不存在可移动媒介，也总是有两个存储空间，而且无论外部空间可否移动，API的行为都保持一致。

内部存储：

  • 总是可用
  • 应用默认可以访问存储的文件
  • 用户卸载应用后，系统移除所有内部存储中的文件

外部存储：

  • 并非总是可用，因为用户可能移除外部存储
  • 全局可读，所以存储的文件可能未经自己控制就被读取
  • 只有使用getExternalFilesDir()存储时，用户卸载应用后，系统才会删掉应用的文件。

外部存储最适合存储不需要访问限制的文件、希望与其他应用共享的文件以及循序用户使用电脑访问的文件。

A Failed Attempt to Implement Html.ImageGetter with Picasso

Posted on 2015-10-10

Problem

Picasso is an very powerful for handling image downloading and caching operations. However, sometimes we need to display an image in a TextView by calling Html.fromHtml, which need a ImageGetter passed as a parameter.

Essentially speaking, ImageGetter is a function interface used to wrap a single function getDrawable(String source). The getDrawable() function is called by Html.fromHtml as a sync process. So getDrawable() could not be executed from the UI thread. And the problem is how to get a Drawable instance using Picasso synchronously, since an async process could not return a Drawable in a the method body of getDrawable().

Android M 运行时权限

Posted on 2015-10-08

从API 23开始，用户将在应用运行时授予权限而非安装时。用户可以在应用的设置屏幕中重新设置权限。

系统权限被分为两类：普通和危险。

  • 普通权限不直接危及用户的隐私。如果应用在manifest中列出普通权限，系统将自动授予。
  • 危险权限会允许应用访问用户的私密数据。如果列出在manifest中，用户必须显示批准权限。

A Problem about notifyDataSetChanged()

Posted on 2015-10-07

Problem

I found a problem when using adapter of a RecyclerView. The notifyDataSetChanged() method did not work because I wrapped it in a recursive method which never returns.

And I found that the notifyDataSetChanged() method could only work after the wrapped method returned. For example, if a sleep is added after notifyDataSetChanged(), it won’t take effect before the sleep is finished.

void wrapper(){
    mAdapter.notifyDataSetChanged();
    try{
        TimeUnit.Seconds.sleep(5);
    }catch(Exception ignored){
    }
}
