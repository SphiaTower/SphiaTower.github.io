Notes on Content Providers (II) Creating a Content Provider

Posted on 2015-11-16

Before You Start Building

You need to build a content provider if you want to provide one or more of the following features:

  • You want to offer complex data or files to other applications.
  • You want to allow users to copy complex data from your app into other apps.
  • You want to provide custom search suggestions using the search framework.

Follow these steps to build your provider:

  1. Design the raw storage for your data. A content provider offers data in two ways: File data or “Structured” data.
  2. Define a concrete implementation of the ContentProvider class and its required methods.
  3. Define the provider’s authority string, its content URIs, and column names. If you want the provider’s application to handle intents, also define intent actions, extras data, and flags. Also define the permissions that you will require for applications that want to access your data. You should consider defining all of these values as constants in a separate contract class; later, you can expose this class to other developers.
  4. Add other optional pieces, such as sample data or an implementation of AbstractThreadedSyncAdapter that can synchronize data between the provider and cloud-based data.

Notes on Content Providers (I) Content Provider Basics

Posted on 2015-11-16

Content Provider Basics

A provider isn’t required to have a primary key, and it isn’t required to use _ID as the column name of a primary key if one is present. However, if you want to bind data from a provider to a ListView, one of the column names has to be _ID.

The ContentResolver object in the client application’s process and the ContentProvider object in the application that owns the provider automatically handle inter-process communication. ContentProvider also acts as an abstraction layer between its repository of data and the external appearance of data as tables.

Notes on the Activity Lifecycle

Posted on 2015-11-16

Starting an Activity

Technically, the activity becomes visible to the user when onStart() is called, but onResume() quickly follows and the activity remains in the Resumed state until something occurs to change that, such as when a phone call is received, the user navigates to another activity, or the device screen turns off.

Most apps don’t need to implement this method because local class references are destroyed with the activity and your activity should perform most cleanup during onPause() and onStop(). However, if your activity includes background threads that you created during onCreate() or other long-running resources that could potentially leak memory if not properly closed, you should kill them during onDestroy().

An Encoding Problem in Clion

Posted on 2015-11-11

Clion is a great IDE of C/C++, made by JetBrain. But I found a problem that when the output of console is Chinese, it can’t be displayed correctly.

I tried to use the same method as Intellij IDEA, changing the encoding from GBK to UTF-8 at the right bottom corner. However, it didn’t work like IDEA 14.

After a lot of attempts modifying different settings, I have found the solution: In File > Settings > Editor > File Encoding, modify both IDE Encoding and Project Encoding into UTF-8. After applied, click the console dock and then change the right bottom corner from UTF-8 to GBK. Then the console can output Chinese now.

EmptyView in ViewPager

Posted on 2015-10-15

Today I encountered a very subtle bug that the Fragments restored from a ViewPager show its empty view with their data already loaded.

My CustomFragment shows an empty view when initialized, and switch to the RecyclerView once data is loaded. The process of animation has been summarized in one of my old articles(http://sphiatower.github.io/2015/09/08/viewpager/). In short, it hides the RecyclerView and shows the empty view in the onCreateView callback.

Every time the data of RecyclerView is updated, it will check an empty flag whether it’s the first time of loading data. If true, the cross fade animation is played to show the content and the flag is updated to remember that data has been loaded.

A Failed Attempt to Implement Html.ImageGetter with Picasso

Posted on 2015-10-10

Problem

Picasso is an very powerful for handling image downloading and caching operations. However, sometimes we need to display an image in a TextView by calling Html.fromHtml, which need a ImageGetter passed as a parameter.

Essentially speaking, ImageGetter is a function interface used to wrap a single function getDrawable(String source). The getDrawable() function is called by Html.fromHtml as a sync process. So getDrawable() could not be executed from the UI thread. And the problem is how to get a Drawable instance using Picasso synchronously, since an async process could not return a Drawable in a the method body of getDrawable().

Posted on 2015-10-08

从API 23开始，用户将在应用运行时授予权限而非安装时。用户可以在应用的设置屏幕中重新设置权限。

  • 危险权限会允许应用访问用户的私密数据。如果列出在manifest中，用户必须显示批准权限。
